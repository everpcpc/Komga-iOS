FAQ

Is KMReader an official Komga app?

No. KMReader is a community-built open-source client that talks to the public Komga APIs. It is designed to feel as polished as a first-party experience, but it is not affiliated with the Komga project.

Can I connect to multiple Komga servers?

Yes. Save as many Komga servers as you like with custom names, switch between them instantly, and manage credentials or cache per server when needed.

What formats can I read?

Comics and manga (DIVINA) are fully supported on all platforms (iOS, iPadOS, macOS, tvOS). EPUB books are supported on iOS/iPadOS/macOS. The comic reader handles every direction, layout, and gesture you expect, while EPUB brings theme, font, and pagination controls with an optional comic-style image boost for illustrated stories. tvOS supports DIVINA reading with remote control navigation.

Do I have to re-download books to read offline?

No. KMReader's three-tier caching system (pages, book files, thumbnails) automatically caches recently opened pages and downloads EPUB files for offline or incognito reading. Adjustable page cache limits, real-time cache size and file count monitoring for each cache type, and individual cache clearing in Settings. Automatic cleanup occurs when limits are exceeded.

Can administrators manage the Komga server?

Yes. With admin permissions, KMReader unlocks metadata editing, library upkeep, maintenance jobs, and quick server insights directly on your device.

How do updates ship?

KMReader is an open-source Swift project. Pull the latest changes from GitHub, rebuild in Xcode, and you’re ready to go. You can also contribute features or translations via pull request.
