FAQ

Is KMReader an official Komga app?

No. KMReader is a community-built open-source client that talks to the public Komga APIs. It is designed to feel as polished as a first-party experience, but it is not affiliated with the Komga project.

Can I connect to multiple Komga servers?

Yes. The server switcher stores as many instances as you like with custom names, last-used timestamps, and admin/user roles. Switch between them instantly, edit credentials, or delete an entry to clear its cached pages, thumbnails, and EPUB files.

What formats can I read?

DIVINA (comic/manga/BD) archives plus EPUB are fully supported. DIVINA gets multiple reading directions, dual-page layouts, zoom, and tap zones. EPUB offers theme, font, pagination, and optional comic-style image rendering for compatible books.

Do I have to re-download books to read offline?

No. KMReader caches recently opened pages and downloads EPUB files for offline or incognito reading automatically. Tune the 512 MB–8 GB page cache slider, check live size/file counters, and clear page, book, or thumbnail caches individually inside Settings.

Can administrators manage the Komga server?

Yes. If your Komga account has admin permissions, KMReader exposes metadata editing, scans, library operations, progress resets, server info, and Komga metrics (disk usage, counts, task analytics) directly on device.

How do updates ship?

KMReader is an open-source Swift project. Pull the latest changes from GitHub, rebuild in Xcode, and you’re ready to go. You can also contribute features or translations via pull request.
